# Documentation Guide

Not every document under `docs/` is current. The active documentation surface is intentionally small so the repository reads as a roadway-derived directional context product rather than a recovery notebook.

## Active Docs

Use these first:

- `workflow/current_workflow_index.md`: active navigation for the final prototype product.
- `workflow/active_workflow.md`: active command surface and output contracts.
- `workflow/roadway_graph_workflow.md`: graph-first roadway workflow.
- `workflow/roadway_graph_directional_context_milestone.md`: final directional-bin context universe milestone.
- `workflow/roadway_graph_context_analysis_plan.md`: next workflow plan for descriptive analysis outputs.
- `methodology/roadway_graph_methodology.md`: core graph-first methodology.
- `methodology/overview_methodology.md`: repository-level methodology posture.
- `methodology/proposal_alignment_growth_plan.md`: controlled proposal-alignment growth plan.
- `design/current_design_index.md`: active design navigation.
- `design/roadway_graph_context_enrichment_plan.md`: context enrichment design record.
- `design/roadway_graph_next_phase_plan.md`: next-phase design plan after the context prototype.
- `design/roadway_graph_rate_and_modeling_readiness_plan.md`: future readiness plan for rates and modeling; planning only, no rates or models created.
- `reports/README.md`: report-folder navigation, including restored signal-centered material and roadway-graph report planning docs.
- `reports/roadway_graph/roadway_graph_methodology_limitations_memo.md`: current roadway-graph methodology and limitations memo.
- `reports/roadway_graph/roadway_graph_figure_inventory_and_specs.md`: figure and exhibit planning specs; no figures generated.
- `reports/roadway_graph/roadway_graph_report_outline.md`: future roadway-graph report outline; not a final report.
- `reports/roadway_graph/roadway_graph_descriptive_report_draft.md`: first roadway-graph descriptive report draft using accepted outputs only.
- `reports/roadway_graph/roadway_graph_figure_index.md`: generated roadway-graph figure index.
- `reports/roadway_graph/roadway_graph_report_qa.md`: report and figure QA summary.

`overview_methodology.md` at the docs root is a convenience pointer to `methodology/overview_methodology.md`; the methodology folder remains the canonical location.

## Archive Docs

Superseded documentation, one-off audits, old proposal package notes, older graph diagnostics, and recovery planning notes were moved to:

`docs/archive/20260519_cleanup/`

Treat archived docs as history or comparison evidence. They are not current workflow authority.

## What Not To Treat As Current Truth

- Archived docs under `docs/archive/`.
- Legacy material under `legacy/`.
- Older signal-centered context enrichment outputs.
- Older directed segment, directionality experiment, upstream/downstream prototype, proposal package, smoke-test, and recovery outputs under `work/archive/`.
- Superseded roadway_graph diagnostics moved under `work/output/roadway_graph/review/history/repo_cleanup_20260519/`.

The current product does not use crash direction fields, does not let context fields redefine upstream/downstream, and does not treat crash `AREA_TYPE` as roadway-level urban/rural truth.

No final polished roadway-graph report has been created yet. Current roadway-graph report materials under `docs/reports/roadway_graph/` include planning docs plus a draft descriptive report, figure index, and report QA.
