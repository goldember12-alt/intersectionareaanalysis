# Reports Documentation

**Status: CURRENT REPORT NAVIGATION.**

`docs/reports/signal_centered/` contains restored prior signal-centered report material. It is useful as historical report context and as a style reference, but it is not the current roadway-graph methodology.

`docs/reports/roadway_graph/` contains all roadway-graph report documentation. The `roadway_graph_*` documents in that folder include planning, memo, draft report, figure index, and report QA documents for the roadway-derived report stage:

- `roadway_graph/roadway_graph_methodology_limitations_memo.md`: bounded methodology and limitations memo for the accepted table package.
- `roadway_graph/roadway_graph_figure_inventory_and_specs.md`: candidate figure and exhibit inventory.
- `roadway_graph/roadway_graph_report_outline.md`: outline for a future roadway-graph report; not a final report draft.
- `roadway_graph/roadway_graph_descriptive_report_draft.md`: first descriptive roadway-graph report draft.
- `roadway_graph/roadway_graph_figure_index.md`: generated figure and exhibit index.
- `roadway_graph/roadway_graph_report_qa.md`: report and figure QA summary.

No final polished roadway-graph report has been created yet. Current figures are draft static descriptive figures generated from accepted descriptive outputs only.

Current roadway-graph report planning must preserve these boundaries:

- no crash direction fields
- no context-derived upstream/downstream relabeling
- no crash rates or AADT-normalized rates
- no models, regressions, or causal claims
- no policy guidance language
- review priority is not danger, risk, or safety-performance ranking
