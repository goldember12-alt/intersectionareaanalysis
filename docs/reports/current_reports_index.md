## Six-Folder Contract

- `docs/design/` = proposed schemas, planning, future designs
- `docs/methodology/` = stable methodological explanations
- `docs/diagrams/` = figure/source diagram assets
- `docs/reports/` = polished/shareable reports
- `docs/results/` = curated result/readout summaries, not raw CSVs
- `docs/workflow/` = active commands, output contracts, and operational notes

## Reports Folder Contract

`docs/reports/` is for polished, forward-facing documents or report-ready markdown/PDF sources that have been manually reviewed or are intended for sharing. For the current roadway-graph stage, it also contains explicitly labeled planning and memo documents that prepare a future report. It should not contain active scratch notes, unreviewed workflow logs, or raw audit logs.
