Working title:

**Roadway-Derived Directional Context for Downstream Functional Area Analysis at Signalized Intersections**

Purpose:

Document the accepted roadway-graph descriptive table package, explain the directional-bin methodology, summarize descriptive context outputs, and define limitations and next review steps before stakeholder-facing interpretation.

## 14. Relationship to Signal-Centered Report

The restored signal-centered report material under `docs/reports/signal_centered/` is useful as a report-style and narrative-sequencing reference. The roadway-graph report should differ in several important ways:

- roadway-derived directional bins replace signal-centered buffer/zone logic
- upstream/downstream comes from roadway graph directionality
- crash direction fields are not used
- context layers enrich bins rather than define upstream/downstream
- descriptive outputs are not policy claims, crash-rate models, or causal findings
- undivided roads remain shared-centerline records by default unless later methodology changes that treatment
