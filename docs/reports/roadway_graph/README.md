The copied stakeholder figure package under `docs/reports/roadway_graph/figures/` is the manually refined report-facing copy. The corresponding generated `/work/output/roadway_graph/report/current/` figures remain preserved as prior generated outputs.

The current report draft uses accepted descriptive outputs and aggregate AADT-normalized prototype summaries only. Unit-level rates remain QA-only and are not included as stakeholder-facing unit-rate tables or figures. Fixed distance-band rates, raw bin-level rates, models, regressions, predictions, causal claims, policy guidance, comparative-performance rankings, and final design recommendations remain out of scope.

The context relationship package is a second report figure package focused on pre-regression descriptive relationships across distance, signal-relative direction, speed, AADT, access density, roadway representation, and crash-level AREA_TYPE context where available. Access density is calculated at local signal-direction profile grain and then summarized, not per 50-ft bin or across the entire displayed group. Rate cells are aggregate prototype cells with denominator, sparse-cell, or review notes where needed.

The context relationship package is generated by:

```powershell
.\.venv\Scripts\python.exe -m src.active.roadway_graph.roadway_graph_context_relationship_figures
```

The internal model technical review memo summarizes the category-simplified crash-count model package for internal technical review only. It uses existing outputs under `work/output/roadway_graph/analysis/current/crash_count_simplified_internal_model/` and supporting review tables under `work/output/roadway_graph/analysis/current/crash_count_internal_model_review/`. It does not fit new models, create stakeholder-facing conclusions, create rankings, make causal claims, or recommend downstream functional area distances.
